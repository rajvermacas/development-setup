# Search Strategies for Tech Research

## Query Optimization Techniques

### 1. Library Discovery Queries

**Pattern: Finding libraries for a specific use case**

```
Best structure: "[language] [task] library" + year filter
Examples:
- "Python PDF manipulation library 2024"
- "JavaScript state management library comparison 2025"
- "Rust async runtime library"

Advanced modifiers:
- Add "comparison" for feature matrices
- Add "benchmark" for performance data
- Add "production-ready" or "mature" to filter stable options
```

**Pattern: Evaluating library quality**

```
Key indicators to search for:
- "[library name] vs [alternative]" - comparative analysis
- "[library name] performance benchmark"
- "[library name] production issues" - real-world problems
- "[library name] maintenance status" - activity check
- "why use [library name]" - community rationale
```

### 2. Documentation Queries

**Pattern: Finding official documentation**

```
Optimal queries:
- "[library name] official documentation"
- "[library name] api reference"
- "[library name] [version] docs"

For specific features:
- "[library name] [feature] documentation"
- "[library name] [feature] example"
- "[library name] migration guide [old version] to [new version]"
```

**Pattern: Finding integration guides**

```
Use these structures:
- "[library A] integrate with [library B]"
- "[framework] [library] setup guide"
- "[library] quickstart tutorial"
```

### 3. Best Practices & Patterns Queries

**Pattern: Architectural patterns**

```
Effective queries:
- "[technology] architecture best practices 2024"
- "[problem domain] design patterns"
- "[technology] production architecture examples"
- "how to structure [type of application]"
- "[technology] scalability patterns"

Domain-specific:
- "microservices communication patterns"
- "React state management patterns"
- "database schema design patterns"
```

**Pattern: Security and performance**

```
Critical areas:
- "[technology] security best practices"
- "[technology] performance optimization"
- "[technology] common vulnerabilities"
- "[technology] production checklist"
```

### 4. Code Examples & Tutorials

**Pattern: Finding quality examples**

```
High-value queries:
- "[library name] real-world example"
- "[library name] tutorial step by step"
- "[specific use case] implementation [technology]"
- "[library name] cookbook" or "recipes"

Filter by recency:
- Add year (2024, 2025) for modern approaches
- Add "latest" or "updated" for current practices
```

## Search Result Evaluation

### Quality Indicators

**Prioritize sources with:**
- Official documentation sites (.io, .dev, .org domains)
- Well-known technical blogs (Martin Fowler, Netflix Tech Blog, etc.)
- Repository README files and wikis (github.com)
- Technical communities (Dev.to, HackerNoon for 2023+)
- Documentation aggregators (ReadTheDocs, npm docs)

**Treat with caution:**
- Content farms with generic advice
- Very old posts (pre-2020 for rapidly evolving tech)
- Marketing-heavy vendor content
- Unverified Stack Overflow answers (check votes/acceptance)

### Source Reliability Matrix

| Source Type | Reliability | Best For |
|-------------|------------|----------|
| Official docs | Highest | API references, features, setup |
| GitHub repos | High | Code examples, real implementations |
| Technical blogs (established) | High | Best practices, patterns, case studies |
| Stack Overflow | Medium-High | Specific problems, edge cases |
| Medium articles | Medium | Tutorials, learning (verify author) |
| Reddit (r/programming, language subs) | Medium | Community opinions, trends |
| Tutorial sites | Variable | Learning basics (check date) |

## Multi-Source Research Strategy

For comprehensive understanding, combine multiple search approaches:

1. **Start broad** - "[topic] overview" or "[topic] landscape"
2. **Narrow to specifics** - "[specific library/approach] deep dive"
3. **Compare options** - "[option A] vs [option B]"
4. **Validate with community** - "[topic] reddit" or "[topic] HackerNews"
5. **Check official sources** - Direct documentation and repos

## Temporal Considerations

### When to use time filters:

- **Require recent (2024-2025):**
  - Rapidly evolving tech (JavaScript frameworks, AI/ML tools)
  - Security best practices
  - Cloud services and APIs
  - Performance benchmarks

- **Accept older (2020+):**
  - Fundamental patterns and principles
  - Stable, mature technologies
  - Computer science fundamentals
  - Design patterns

- **Any age acceptable:**
  - Mathematical algorithms
  - Core language features
  - Foundational concepts

## Advanced Search Operators

Use these for precision:

```
site:github.com "[library name]" example
site:docs.[technology].com "[feature]"
"[exact phrase]" -tutorial (exclude tutorials)
intitle:"[library name]" best practices
filetype:pdf "[topic]" architecture
```
